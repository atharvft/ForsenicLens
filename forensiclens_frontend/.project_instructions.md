# ForensicLens AI - Project Instructions

## Overview
Forensic photo analysis web application built for a hackathon. NextJS 14 frontend that connects to a FastAPI backend running on Google Colab via ngrok.

## Tech Stack
- NextJS 14, React 18, Tailwind CSS 3
- NextAuth.js v4 (credentials-based auth)
- Prisma ORM with PostgreSQL
- AWS S3 for cloud storage
- Framer Motion for animations

## Visual Style
- **Theme:** Dark futuristic/tech with neon cyan (#00F0FF) accent
- **Background:** #0a0a1a (near-black with blue tint)
- **Cards:** #0f0f2a/80 with cyan border accents
- **Fonts:** Inter (body), Orbitron (headings/branding)
- **Effects:** Neon glow, grid background, gradient borders, blur backdrops
- **CSS Classes:** `.neon-glow`, `.neon-text`, `.neon-border`, `.grid-bg`

## Pages
1. **/** - Landing page (redirects to /dashboard if authenticated)
2. **/login** - Email/password login
3. **/register** - Registration with name/email/password
4. **/dashboard** - Stats cards, quick actions, recent photos
5. **/upload** - Drag-and-drop photo upload with S3 presigned URLs
6. **/history** - Paginated photo list with search/filter
7. **/photo/[id]** - Photo detail with anomaly detection, upscaling, side-by-side comparison, zoom, download

## Backend Integration
- `NEXT_PUBLIC_API_URL` env var points to ngrok URL
- `lib/api-client.ts` handles backend communication with JWT tokens stored in localStorage
- Backend endpoints: /api/auth/register, /api/auth/login, /api/photos/upload, /api/photos/{id}/detect-anomalies, /api/photos/{id}/upscale
- Processing includes fallback demo simulation if backend unavailable

## Database Schema
- **User:** id, email, name, password, timestamps
- **Photo:** id, userId, originalCloudPath/Url, anomalyCloudPath/Url, upscaledCloudPath/Url, fileName, fileSize, mimeType, status, anomalyStatus, upscaleStatus, anomalyData, upscaleData, backendPhotoId, timestamps

## Auth
- NextAuth credentials provider with bcryptjs hashing
- Middleware protects /dashboard, /upload, /history, /photo routes
- Auth options in `lib/auth-options.ts`

## Forensic Analysis Features (Photo Detail Page)
- **AI Analysis** (`/api/photos/[id]/analyze`) - GPT-4.1 forensic analysis with aggressive AI detection, EXIF metadata extraction via exifr, metadata provenance scoring
- **EXIF Metadata** (`/api/photos/[id]/metadata`) - Dedicated EXIF extraction endpoint
- **AI Upscaling** (`/api/photos/[id]/upscale`) - Two-step: gpt-4.1 description → gpt-5.1 image generation (model name is gpt-5.1, NOT flux)
- **DOCX Report** (`/api/photos/[id]/report`) - Professional forensic report download using `docx` package
- **Reverse Image Search** - Opens Google Lens, TinEye, Yandex, Bing Visual Search in new tabs
- **Chain of Custody Timeline** - Visual timeline built from EXIF + system data
- **Heatmap Overlay** - Canvas-based anomaly visualization color-coded by severity
- **Social Media Tracker** - Manual platform links for cross-platform image searching
- **JSON/CSV Export** - Download structured analysis data in either format

## Key Patterns
- Server components as page wrappers, client components for interactivity
- Optional chaining everywhere for crash prevention
- Polling (3s interval) for processing status updates
- Toast notifications via react-hot-toast
- Packages: docx, file-saver for report generation
